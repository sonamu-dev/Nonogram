# UX (Draft)

## 화면 흐름
- `Play`: 현재 퍼즐 플레이, 힌트 확인, Undo/Redo, 진행 저장
- `Library`: 저장된 퍼즐/최근 플레이/난이도별 목록 진입
- `Generate`: 크기/옵션 선택 후 생성, 진행률/취소 제공
- `Settings`: 입력 방식, 접근성 옵션, 시각/동작 기본값 설정

## 입력 규칙
- 마우스:
  - 좌클릭 채움, 우클릭 표시(X), 드래그 연속 입력 지원
- 키보드:
  - 방향키 이동, 단축키로 채움/표시/해제, Undo/Redo 지원
- 입력 모드는 항상 현재 상태가 시각적으로 명확해야 한다.

## 실수 방지 UX
- 파괴적 동작(초기화/덮어쓰기)은 확인 단계를 둔다.
- 충돌 가능 입력(명백한 모순)은 경고하되 강제 차단 여부는 설정 가능하게 둔다.
- 자동 저장과 복구 진입점을 기본 제공한다.

## Fluent 스타일 시각/상태 원칙
- 최신 Windows Fluent 컴포넌트와 토큰을 우선 사용한다.
- 상태 표현은 색상만 의존하지 않고 아이콘/텍스트를 병행한다.
- 로딩/생성/분석 중에는 진행률 + 취소 액션을 항상 노출한다.
- 고대비/배율 환경에서도 정보 계층이 유지되도록 설계한다.

## Play MVP (Stage8)
- `Play` 화면에서 `New Puzzle` 버튼으로 퍼즐을 즉시 생성할 수 있어야 한다.
- 생성 중에는 `ProgressRing`과 `Cancel` 버튼을 노출해 UI가 멈추지 않도록 한다.
- 행/열 클루와 셀 보드를 동시에 보여주고 셀 클릭 입력(Unknown -> Filled -> Empty -> Unknown)을 지원한다.
- 생성 결과에 정답 보드가 포함되면 현재 보드와 정답 일치 시 `Solved!` 상태를 표시한다.

## Play UX v1 (Stage9)
- 입력 모드(`Fill / MarkEmpty / Erase`)를 `Play` 상단에서 명확하게 전환할 수 있어야 한다.
- 좌클릭/드래그는 현재 모드를 연속 적용하고, 우클릭은 `MarkEmpty`를 즉시 적용한다.
- 드래그 1회 입력은 Undo 1회로 되돌릴 수 있어야 한다.
- `Undo/Redo/Reset`은 항상 즉시 반영되고, 실행 직후 `Solved` 판정이 다시 계산되어야 한다.
- 셀 상태는 Unknown(빈칸), Filled(채움), Empty(X)로 색/기호를 함께 사용해 구분한다.

## 핵심 결론
- UX 목표는 “빠른 입력”과 “실수 복구 가능성”의 균형이다.
- Play/Library/Generate/Settings 흐름은 단순하고 예측 가능해야 한다.
- Fluent 원칙은 시각 일관성보다 상태 전달 명확성을 우선한다.
