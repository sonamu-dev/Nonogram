# Requirements (Draft)

## 기능 요구사항

### 1) 플레이
- 사용자는 퍼즐 보드에서 셀을 채움/표시(X)/해제할 수 있어야 한다.
- 행/열 힌트를 기반으로 현재 진행 상태를 확인할 수 있어야 한다.
- 완료 조건 충족 시 성공 상태를 명확히 보여줘야 한다.

### 2) 입력
- 마우스(클릭/드래그)와 키보드 입력을 모두 지원해야 한다.
- 입력 모드(채움/표시) 전환이 즉시 가능해야 한다.

### 3) Undo/Redo
- 사용자의 입력 이력을 되돌리기/재실행할 수 있어야 한다.
- 연속 입력(드래그)은 의미 단위로 묶어 처리할 수 있어야 한다.

### 4) 저장/복원
- 진행 중 퍼즐 상태를 로컬에 저장하고 재시작 시 복원할 수 있어야 한다.
- 최근 플레이 목록 또는 마지막 플레이를 빠르게 이어서 할 수 있어야 한다.

### 5) 퍼즐 생성
- 사용자는 크기/옵션을 선택해 새 퍼즐을 생성할 수 있어야 한다.
- 생성 작업은 UI를 멈추지 않아야 하며 취소 가능해야 한다.

### 6) 난이도 표시
- 퍼즐마다 난이도 등급(Easy~Expert)을 제공해야 한다.
- 등급과 함께 간단한 근거 요약을 확인할 수 있어야 한다.

## 비기능 요구사항

### 성능/응답성
- 일반 입력은 즉시 반응해야 하며 UI 프레임 저하를 최소화한다.
- 장시간 작업(생성/분석)은 백그라운드 실행 + 진행률/취소를 제공한다.

### DPI/화면 대응
- 고해상도 DPI 환경에서 텍스트/보드/컨트롤이 깨지지 않아야 한다.
- 창 크기 변경 시 레이아웃이 안정적으로 재배치되어야 한다.

### 접근성
- 키보드 중심 조작, 포커스 이동, 스크린리더 기본 호환을 보장한다.
- 상태 정보(오류/완료/선택)를 시각 외 수단으로도 전달해야 한다.

### 오프라인
- 핵심 기능(플레이/저장/생성/난이도 확인)은 오프라인에서 동작해야 한다.

### 로깅/진단
- 오류와 핵심 사용자 작업 흐름을 로컬 로그로 추적 가능해야 한다.
- 개인 정보는 최소 수집/저장 원칙을 따른다.

## 핵심 결론
- 사용자 가치는 `플레이 안정성 + 생성 품질 + 난이도 신뢰성`이다.
- 성능 원칙은 `비차단 UI + 취소 가능한 장시간 작업`이다.
- 제품 완성도는 `접근성/오프라인/로깅` 기본 충족을 전제로 한다.
