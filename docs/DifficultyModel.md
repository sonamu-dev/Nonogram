# Difficulty Model (Draft)

## 난이도 등급 정의
- `Easy`: 기본 단일 규칙 위주로 해결 가능, 추측 불필요
- `Normal`: 복수 기본 규칙 조합 필요, 짧은 탐색 가능
- `Hard`: 고급 규칙/깊은 연쇄 추론 필요, 제한적 추측 가능
- `Expert`: 고급 추론 + 높은 탐색 비용/백트래킹 발생

## 난이도 산정 지표
- 전략 로그 복잡도: 사용된 규칙 종류와 난도 가중치
- 추측 여부: 순수 논리 해결인지, 분기(guess)가 필요한지
- 백트래킹 비용: 분기 수, 최대 깊이, 실패 분기 비율
- 풀이 비용: 단계 수, 계산 시간, 상태 전환량
- 안정성 지표: 동일 퍼즐 반복 분석 시 등급 일관성

## 산정 결과 구조(초안)
- 입력: 퍼즐 정의 + Solver 실행 로그
- 출력:
  - 최종 등급(Easy~Expert)
  - 총점(내부용)
  - 근거 항목(상위 2~4개)

## 사용자 노출 설명 포맷
- 표시 예시:
  - `난이도: Hard`
  - `근거: 고급 규칙 3회, 추측 1회, 백트래킹 깊이 4`
  - `요약: 논리만으로는 후반부가 어려워 제한적 탐색이 필요합니다.`

## 핵심 결론
- 난이도는 “정답 유무”가 아니라 “해결 과정 비용”으로 정의한다.
- 설명 가능한 난이도를 위해 전략 로그 표준화가 선행되어야 한다.
- 사용자에게는 등급 + 짧은 근거를 일관된 형식으로 제공한다.
